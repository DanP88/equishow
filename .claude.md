2. Run `npm start` to verify the server is working

## Important Features & Workflows

### Reservation Flow
- **Coach**: Services tab > Select coach > `reserver-coach.tsx` → Modal confirmation
- **Transport**: Services tab > Select transport > `reserver-transport.tsx` → Modal confirmation
- **Box**: Services tab > Select box > `reserver-box.tsx` → Modal confirmation

**Modal Pattern**: Show confirmation instead of Alert, display details, return to Services tab

### User Roles
- **cavalier** (rider): Can view services, book sessions, view profiles
- **coach** (instructor): Can create annonces, manage schedule, view students
- **organisateur** (organizer): Can create events, manage services

### Data Stores
- `chevauxStore`: Horse data
- `userStore`: Current user profile
- `coachesStore`: Coach profiles
- `coachAnnoncesStore`: Coach announcements
- `transportsStore`: Transport offers
- `boxesStore`: Box offers
- `notificationsStore`: Notifications

## Debugging Tips

### Server Won't Start
- Clear cache: `rm -rf .expo node_modules/.cache metro-cache`
- Reinstall: `npm install --legacy-peer-deps`
- Check port 8081 is free

### App Timeout in Expo Go
- Verify same Wi-Fi network (phone & Mac)
- Check no VPN active
- Restart Expo server
- Clear Expo Go cache on device

### TypeScript Errors
- Check types in `/types/` folder
- Update after schema/data changes
- Use `any` sparingly for mock data
